1. Prerequisites
Ensure you have the following installed locally on your development environment:
Python 3.10 or higher
Docker Desktop (for running MongoDB locally)
Ollama App
2. Set Up the Infrastructure
Spin up a local containerized instance of MongoDB using Docker:
Bash
docker run -d -p 27017:27017 --name local-mongo mongo:latest
Pull and run your local open-source LLM using Ollama in your terminal:
Bash
ollama run llama3
3. Clone and Setup Environment
Navigate to your project directory, set up a python virtual environment, and install your dependencies:
Bash
# Set up virtual environment
python3 -m venv venv
source venv/bin/activate

# Install required frameworks
pip install fastapi uvicorn motor langchain_ollama pydantic pypdf streamlit requests
4. Running the Applications
To run this distributed multi-tier app, launch both services simultaneously in separate terminal panels:
Terminal 1: Start the FastAPI Backend Application
Bash
source venv/bin/activate
uvicorn main:app --reload
The interactive Swagger documentation interface will be accessible at: http://127.0.0.1:8000/docs
Terminal 2: Start the Streamlit Front-End Interface Dashboard
Bash
source venv/bin/activate
streamlit run app.py
The user interface web app will automatically launch at: http://localhost:8501
📊 Core Technical Implementations Highlighted
Intelligent Dynamic Deserialization: Rather than writing manual text parsing regex patterns, this code forces an open-source LLM to read messy, unstructured human medical text layouts and cast them directly into strongly-typed validation objects (acting like a Jackson object mapper in Java ecosystem).
Graceful Error Resilience: If a background extraction task hits an internal structural edge-case error, the engine safely intercepts the exception context, captures the traceback logs inside MongoDB, drops the dirty system files, and flips the client tracking state to "FAILED" rather than terminating or crashing system server threads.
